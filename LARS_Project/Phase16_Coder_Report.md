# Phase 16 Coder Report — DP Print Format View 완료

## 1. 개요
생산 현장에서 실제 사용되는 엑셀 양식과 동일한 레이아웃을 웹에서 제공하기 위해 "Print View" 기능을 구현하였습니다. 라인별, 모델별 필터링 기능과 함께 A3 Landscape 인쇄에 최적화된 고정 헤더 테이블을 지원합니다.

## 2. 작업 상세

### 2.1 백엔드 데이터 확장 (Task 16-A)
- **API 수정**: `/api/v1/dp/lots-raw` 응답 데이터에 `input_qty`(실적 투입량) 및 `output_qty`(실적 생산량) 필드를 추가하였습니다.
- **JOIN 최적화**: `ProductionLine` 테이블과의 조인을 통해 `line_code`를 안정적으로 확보하여 반환합니다.

### 2.2 프론트엔드 인터페이스 확장 (Task 16-B)
- **타입 정의**: `LotRaw` 인터페이스를 확장하여 실적 데이터(`input_qty`, `output_qty`)를 처리할 수 있도록 보완하였습니다.

### 2.3 전용 프린트 뷰 컴포넌트 개발 (Task 16-C)
- **`DailyPlanPrintView.tsx`**: 
    - **계층형 헤더**: 엑셀 양식과 동일하게 [투입시점 | W/O | 부품번호 | 수량(계획/IN/OUT) | 날짜별 수량]으로 이어지는 3단 고정 헤더를 구현하였습니다.
    - **인터랙티브 필터**: 상단에 라인 탭(ALL/C11/C5 등)과 모델별 Pill 버튼을 배치하여 출력 대상을 정밀하게 제어할 수 있습니다.
    - **자동 요약**: 헤더 하단(Row 3)에 현재 필터링된 데이터의 합계(Sum)를 실시간으로 계산하여 표시합니다.
    - **정렬**: 모든 데이터는 투입시점(PST) 오름차순으로 정렬되어 생산 순서를 직관적으로 보여줍니다.

### 2.4 사용자 인터페이스 통합 (Task 16-D)
- **탭 전환 시스템**: `DailyPlanPage` 우측 상단에 "Raw Table"과 "Print View" 탭을 추가하여 사용 목적에 따라 화면을 즉시 전환할 수 있습니다.
- **상태 보존**: 사용자가 선택한 DP 배치는 탭을 전환해도 유지되도록 설계되었습니다.

### 2.5 인쇄 최적화 (Task 16-E)
- **CSS 미디어 쿼리**: `@media print` 설정을 통해 인쇄 시 사이드바, 헤더, 필터 컨트롤을 자동 숨김 처리합니다.
- **A3 Landscape**: 대량의 날짜 컬럼을 한 페이지에 담기 위해 A3 가로 방향 출력을 기본 설정으로 적용하였습니다.

## 3. 검증 결과
- **백엔드**: `/dp/lots-raw` 호출 시 `input_qty`, `output_qty` 포함 확인 완료.
- **프론트엔드**: TypeScript 오류 0건, 빌드 및 배포 완료.
- **동작**: 라인별 필터링 시 모델 목록 동적 갱신 및 합계 행 자동 계산 정상 작동 확인.

## 4. 향후 과제
- 현재 LARS 시스템에 적재되지 않는 엑셀의 `Connecter`, `TPL` 등 메타 데이터는 공란 또는 제외 처리하였으며, 추후 데이터 확충 시 추가 렌더링이 가능하도록 구조를 설계하였습니다.
