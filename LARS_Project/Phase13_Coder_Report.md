# Phase 13 Coder Report — ItemMaster 구매품/사내품 분리 및 전 문서 컬럼 필터 도입 완료

## 1. 개요
ItemMaster의 관리 효율성을 높이기 위해 Vendor 유무에 따른 탭 분리 기능을 도입하고, 시스템 전반의 데이터 테이블에 사용자 맞춤형 컬럼 필터 시스템을 구축하였습니다.

## 2. 작업 상세

### 2.1 범용 컬럼 필터 시스템 (Task 13-A)
- **`useColumnFilter` 훅**: 데이터 배열을 입력받아 컬럼별 텍스트 매칭 필터링을 수행하는 로직을 모듈화하였습니다. 다중 필터 및 초기화 기능을 지원합니다.
- **`FilterableHeader` 컴포넌트**: 테이블 헤더(`th`)에 필터 아이콘과 입력창을 통합한 재사용 컴포넌트를 제작하였습니다. 필터 활성 시 시각적 강조 효과를 제공합니다.

### 2.2 ItemMaster 고도화 (Task 13-B)
- **Vendor 기반 탭 분리**: `vendor_raw` 데이터의 존재 여부에 따라 '구매품'과 '사내생산품' 탭으로 분류하였습니다. 
- **탭별 특화 UI**: 사내생산품 탭에서는 관리 대상이 아님을 알리는 안내 문구를 배치하고, 각 탭의 데이터 건수를 실시간으로 표시합니다.
- **컬럼 필터 적용**: Lv, 품번, 품명, 협력사 컬럼에 필터를 적용하여 수만 건의 품목 중 필요한 데이터를 즉시 찾을 수 있습니다.

### 2.3 전사적 필터 확산 (Task 13-C, 13-D, 13-F)
- **Part List**: 품번, 품명, 협력사 기준 필터 추가.
- **Daily Plan**: W/O 번호, 모델명 기준 필터 추가.
- **BOM List**: 설명, 버전 기준 필터 추가.

### 2.4 BOM 트리 검색 최적화 (Task 13-E)
- **Flat Search 모드**: 트리 구조에서의 검색 복잡도를 해결하기 위해, 검색어 입력 시 즉시 Flat한 리스트로 전환하여 매칭된 모든 부품을 표시하고 검색 종료 시 원래의 트리 구조로 복구되는 하이브리드 검색 방식을 도입하였습니다.

## 3. 검증 결과
- **정적 분석**: TypeScript 속성 매핑 오류 수정 후 `npx tsc --noEmit` 결과 오류 0건.
- **빌드**: Vite 빌드 성공 및 런타임 동작 확인.
- **UX**: 모든 페이지에서 필터 아이콘 클릭 및 실시간 필터링이 정상 작동하며, '필터 초기화' 버튼을 통해 원복이 가능함을 확인하였습니다.

## 4. 특이사항
- `useColumnFilter`의 반환값 명칭을 `onFilter`, `onToggle`로 조정하여 컴포넌트 Props와 일치시킴으로써 코드 가독성과 타입 안정성을 확보하였습니다.
