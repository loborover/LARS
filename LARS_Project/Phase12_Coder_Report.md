# Phase 12 Coder Report — BOM List 그룹핑 및 트리 뷰어 개선 완료

## 1. 개요
BOM 모델 식별 체계(Model + Suffix)를 사용자 UI에 직관적으로 반영하기 위해, 모델 목록의 그룹화 기능과 계층 구조를 시각적으로 탐색할 수 있는 인터랙티브 트리 뷰어를 구현하였습니다.

## 2. 작업 상세

### 2.1 BOM 목록 그룹화 (Task 12-A)
- **Model Grouping**: `BOMListPage.tsx`에서 `model_code`가 동일한 여러 Suffix(Variant)를 하나의 그룹으로 묶어 표시합니다.
- **Collapsible Rows**: 그룹 헤더를 클릭하여 하위 Variant 리스트를 접거나 펼칠 수 있습니다.
- **UI 최적화**: 단일 Variant만 존재하는 경우 그룹 헤더 없이 바로 표시하여 불필요한 클릭을 줄였습니다.

### 2.2 인터랙티브 BOM 트리 뷰어 (Task 12-B)
- **Hierarchical Building**: Flat한 부품 리스트를 `level`과 `path` 기반의 트리 구조로 변환하는 `buildTree` 알고리즘을 구현하였습니다.
- **Recursive Rendering**: `TreeRow` 컴포넌트를 재귀적으로 호출하여 무제한 깊이의 트리를 렌더링합니다.
- **토글 기능**: 자식이 있는 노드에 ▶/▼ 버튼을 배치하여 개별 노드 단위로 접기/펼치기가 가능합니다. (Lv 2 이상 기본 접힘 설정)
- **전역 컨트롤**: '전체 펼치기' 및 '전체 접기' 버튼을 추가하여 대규모 BOM 탐색 편의성을 높였습니다.

### 2.3 상세 페이지 정보 보강 (Task 12-C)
- **헤더 개선**: `BOMDetailPage.tsx` 상단 카드에서 `model_code`와 `suffix`를 분리하여 강조 표시하고, 해당 모델의 총 부품 수를 시각화하였습니다.

## 3. 검증 결과
- **UI/UX**: 그룹핑 및 트리 토글 동작이 부드럽게 작동하며, 대량 데이터(Variant 100개 이상)에서도 성능 저하 없이 렌더링됨을 확인하였습니다.
- **정적 분석**: TypeScript 오류 0건 및 Vite 빌드 성공.

## 4. 특이사항
- 대체 부품(Substitute)의 경우 `level < 0`으로 판단하여 노란색 배경과 '대체' 배지를 표시하도록 시각적 예외 처리를 적용하였습니다.
