# 업무 참조 가이드 (Work Reference Guide)

> 이 문서는 에이전트가 업무를 수행할 때 참고하는 **운영 매뉴얼**입니다.
> 프로젝트에 종속되지 않는 범용 규칙과, 현재 프로젝트의 문서 목록을 함께 관리합니다.

---

## 1. 업무 디렉토리 규칙

- **업무 디렉토리**: `.agent/Work/` — 모든 업무 관련 문서는 이 디렉토리에서 생성·참조·수정합니다.
- **절대 규칙**: 업무 문서를 `.agent/` 루트나 프로젝트 소스 폴더에 만들지 않습니다.
- **새 업무 문서 생성 시**: `.agent/Work/[문서명].md` 경로를 사용합니다.
- **네이밍 규칙**: 문서명은 내용을 한눈에 알 수 있도록 명확하게 짓습니다.

---

## 2. 현재 업무 문서 목록

| 파일명 | 역할 | 갱신 시점 |
|--------|------|----------|
| `Work_Guide.md` | 업무 참조 가이드 (이 파일). 디렉토리 규칙, 문서 목록, 참조 순서. | 문서 추가/삭제 시 |
| `LARS_Project.md` | LARS 프로젝트 개요, 목적, 기술 스택, 모듈 구성. | 프로젝트 방향 변경 시 |

| `Csharp_Review.md` | C# 코드베이스 아키텍처 리뷰. 레이어별 구조, 서비스 분석, VBA 이관 상태. | C# 코드 변경·빌드 검증 후 |

| `Issues.md` | 이슈 트래커. 버그·장애·조사 필요 사항 관리. | 이슈 발생/해결 시 |

---

## 3. 업무 수행 시 참조 순서

1. **새 작업 시작 전**: `.agent/Work/` 디렉토리의 관련 문서를 먼저 읽어 현재 상태를 파악합니다.
2. **프로젝트 맥락 파악**: `LARS_Project.md`를 읽어 전체 방향을 이해합니다.
3. **VBA 관련 질문**: `VBA_Review.md`를 참조하여 원본 로직을 확인합니다.
4. **C# 구현 관련**: `Csharp_Review.md`를 참조하여 현재 구현 상태를 확인합니다.
5. **개발 계획/우선순위**: `Migration_Plan.md`를 참조합니다.
6. **작업 완료 후**: 변경사항을 해당 업무 문서에 반영·업데이트합니다.

---

## 4. 문서 갱신 원칙

- C# 코드를 수정하면 → `Csharp_Review.md` 관련 섹션을 업데이트합니다.
- VBA 분석이 추가되면 → `VBA_Review.md`를 보완합니다.
- 스프린트를 완료하면 → `Migration_Plan.md`의 진행 상태를 체크합니다.
- 새로운 업무 문서가 추가되면 → 이 파일(§2)의 테이블을 업데이트합니다.
- 프로젝트 방향이 바뀌면 → `LARS_Project.md`를 업데이트합니다.
