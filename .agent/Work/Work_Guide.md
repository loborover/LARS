# 업무 참조 가이드 (Work Reference Guide)

> 이 문서는 에이전트가 업무를 수행할 때 참고하는 **운영 매뉴얼**입니다.
> 프로젝트에 종속되지 않는 범용 규칙과, 현재 프로젝트의 문서 목록을 함께 관리합니다.

---

## 1. 문서 체계 개요

에이전트의 행동을 결정하는 문서는 다음 3계층으로 구분됩니다.

| 계층 | 파일 | 역할 | 성격 |
|------|------|------|------|
| **정체성** | `.agent/Identity.md` | 에이전트의 R&R, 소통 원칙, 운영 프로세스 | 프로젝트 **비종속** (범용 프롬프트) |
| **업무방식** | `.agent/Work/Work_Guide.md` (이 파일) | 업무 디렉토리 규칙, 문서 목록, 참조 순서 | 프로젝트 **비종속** (범용 가이드) |
| **프로젝트** | `.agent/Work/*_Project.md` | 프로젝트별 기획, 기능, 기술 스택, 아키텍처 | 프로젝트 **종속** (각 프로젝트마다 다름) |

---

## 2. 업무 디렉토리 규칙

- **업무 디렉토리**: `.agent/Work/` — 모든 업무 관련 문서는 이 디렉토리에서 생성·참조·수정합니다.
- **절대 규칙**: 업무 문서를 `.agent/` 루트나 프로젝트 소스 폴더에 만들지 않습니다.
- **새 업무 문서 생성 시**: `.agent/Work/[문서명].md` 경로를 사용합니다.
- **네이밍 규칙**: 문서명은 내용을 한눈에 알 수 있도록 명확하게 짓습니다.

---

## 3. 표준 업무 문서 유형

모든 프로젝트에서 아래 유형의 문서를 `.agent/Work/` 하위에 관리합니다.

| 문서 패턴 | 역할 | 갱신 시점 |
|----------|------|----------|
| `Work_Guide.md` | 업무 참조 가이드 (이 파일). 디렉토리 규칙, 문서 목록, 참조 순서. | 문서 추가/삭제 시 |
| `*_Project.md` | **프로젝트 기획서**. 목적, 핵심 기능, 기술 스택, 아키텍처, 유지보수 프로세스. | 프로젝트 방향 변경 시 |
| `*_Review.md` | **코드 리뷰**. 현재 코드베이스의 아키텍처 분석, 서비스 구조, 개선 사항. | 빌드 완료 후 |
| `Migration_Plan.md` | 이관/개발 계획. 스프린트 단위 작업 목록 및 진행 상태. | 스프린트 완료 시 |
| `Issues.md` | 이슈 트래커. 버그·장애·조사 필요 사항 관리. | 이슈 발생/해결 시 |

---

## 4. 업무 수행 시 참조 순서

1. **새 작업 시작 전**: `.agent/Work/` 디렉토리의 관련 문서를 먼저 읽어 현재 상태를 파악합니다.
2. **프로젝트 맥락 파악**: `*_Project.md`를 읽어 전체 방향을 이해합니다.
3. **현재 구현 상태 확인**: `*_Review.md`를 참조하여 코드 구조를 파악합니다.
4. **개발 계획/우선순위**: `Migration_Plan.md`를 참조합니다.
5. **작업 완료 후**: 변경사항을 해당 업무 문서에 반영·업데이트합니다.

---

## 5. 문서 갱신 원칙

- 코드를 수정하면 → `*_Review.md` 관련 섹션을 업데이트합니다.
- 스프린트를 완료하면 → `Migration_Plan.md`의 진행 상태를 체크합니다.
- 새로운 업무 문서가 추가되면 → 이 파일(§3)의 테이블을 업데이트합니다.
- 프로젝트 방향이 바뀌면 → `*_Project.md`를 업데이트합니다.
