# AR MVP 명세서

> 마지막 갱신: 2026-04-05

## 1. MVP 목표

웹 브라우저에서 다음 흐름이 동작하면 MVP로 간주합니다.

1. 사용자가 파일 업로드
2. 서버가 작업 접수
3. Worker가 문서 처리
4. 결과물과 metadata 저장
5. 사용자가 결과 조회 및 PDF 다운로드

## 2. MVP 범위

- 로그인
- 조직/권한 기반 접근 제어의 최소 구현
- BOM 처리
- DailyPlan 처리
- PartList 처리
- ItemCounter 실행
- MultiDocument 매칭
- 작업 이력 조회
- PDF 다운로드
- 메타데이터 기본 검색

## 3. MVP 필수 화면

- 로그인 화면
- 파일 업로드 화면
- 작업 목록/상태 화면
- 작업 상세 화면
- 결과 다운로드 화면

## 4. MVP 필수 API 범주

- 인증 API
- 파일 업로드 API
- 작업 생성 API
- 작업 상태 조회 API
- 결과 목록/다운로드 API
- 메타데이터 검색 API

## 5. MVP 제외 범위

- 고급 시각화 대시보드
- 자연어 입력 기반 보고서 생성
- AI 챗봇 정식 기능
- 고급 Drawing Engine
- 고급 스케줄러
- 외부 시스템 양방향 연동

## 6. MVP 성공 조건

- 사용자가 브라우저만으로 업로드/실행/조회 가능
- 결과와 로그가 저장된다
- 실패한 작업도 원인 추적이 가능하다
- 최소 50명 규모 사용을 고려한 구조가 준비된다
- 레거시 결과와 비교 가능한 기본 검증 경로가 존재한다
