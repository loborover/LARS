# LARS 프로젝트 기획서

> 마지막 갱신: 2026-04-05
> 현재 Source of Truth 문서

## 1. 프로젝트 정체성

### 1.1. AR (AutoReport)
AR은 생산/물류 실무자가 사용하는 `보고서 작성 및 조회 자동화 서비스`입니다.
주요 책임은 다음과 같습니다.

- BOM, DailyPlan, PartList 파일 업로드 및 판별
- 파일 유효성 검사
- 문서 가공 및 정규화
- ItemCounter 집계
- MultiDocument 매칭
- PDF 및 결과물 생성
- 보고서 metadata 기록

### 1.2. LARS
LARS는 AR을 포함하는 상위 플랫폼입니다.
주요 책임은 다음과 같습니다.

- AR 결과 기반 물류 이동 시각화
- 작업 이력 및 메타데이터 검색
- AI chatbot 연동
- 자연어 기반 보고서 조회/생성
- 향후 외부 시스템 연동

## 2. 최종 목표

- Windows, Linux, macOS 어디서나 접근 가능한 웹 기반 시스템
- 사내 작업자와 협력사 작업자가 동시에 사용하는 구조
- 최소 50명 이상의 동시 사용자 대응
- AI가 metadata를 읽고 쓰며 자동화에 참여할 수 있는 구조
- 레거시 VBA 비즈니스 로직은 유지하되, 플랫폼 구조는 현대적으로 재설계

## 3. 현재 저장소 기준 현실 인식

- `VBA/`에는 원본 업무 로직 자산이 존재합니다.
- 저장소 루트에는 `LARS.sln`, `LARS.csproj`, `App.xaml`, `Views/`, `ViewModels/`, `Services/` 등 WPF/C# 자산이 존재합니다.
- `TestSet/`에는 입력 파일, 검증 산출물, 간단한 `TestHarness`가 존재합니다.
- 따라서 현재 저장소는 `레거시 참고 자산 + 검증용 샘플 자산 저장소`로 해석해야 합니다.
- 새 서버형 구현이 아직 본격 착수되지 않았으므로, 문서의 역할은 목표 구조를 고정하고 구현 우선순위를 안내하는 것입니다.

## 4. 목표 기술 방향

- Frontend: `Next.js`
- Backend API: `FastAPI`
- Async Worker: `Celery`
- Queue/Broker: `Redis`
- Database: `PostgreSQL`
- File Storage: NAS 또는 로컬 스토리지, 이후 S3/MinIO 확장 가능
- Deployment: `Docker Compose` 시작

## 5. 핵심 원칙

- 데스크톱 앱 중심 설계를 폐기하고 API 중심 설계를 채택합니다.
- 파일 자체보다 `구조화된 metadata`를 우선 자산으로 봅니다.
- 무거운 문서 처리는 Worker가 담당하고, 웹 요청은 가볍게 유지합니다.
- 권한, 감사 로그, 조직/협력사 구분을 초기에 설계합니다.
- AI는 마지막 부가 기능이 아니라 초기부터 고려되는 확장 축입니다.

## 6. 주요 사용자

- 사내 물류 작업자
- 생산/자재 관리 담당자
- 협력사 작업자
- 관리자
- 향후 AI Agent

## 7. 제품 경계

### AR 서비스 범위
- 문서 업로드
- 문서 판별 및 변환
- PDF 생성
- 작업 상태 관리
- 결과 조회
- 메타데이터 기록

### LARS 플랫폼 범위
- AR 결과 통합 검색
- 시각화 대시보드
- 자연어 질의응답
- 작업 추천 및 AI 보조
- 외부 시스템 연동

## 8. 단계별 제품 전략

### Step 1. AR Web MVP
- 웹에서 업로드, 처리, 결과 조회, PDF 다운로드 제공
- 구조화된 메타데이터 저장
- 조직/사용자 단위 접근 제어의 최소 구현

### Step 2. 운영형 AR
- 감사 로그, 백업, 장애 대응, 모니터링 강화
- 조직/협력사 권한 고도화
- 검색성과 재처리 운영성 확보

### Step 3. LARS 확장
- 시각화 대시보드
- 자연어 조회
- AI metadata read/write
- 외부 시스템 연동

## 9. 성공 기준

- AR MVP가 웹에서 업로드/처리/조회/다운로드까지 동작한다.
- 메타데이터가 DB에 구조화되어 저장된다.
- 50명 규모의 동시 사용자 환경에서도 운영 구조가 무너지지 않는다.
- 이후 LARS 상위 기능이 AR 위에 자연스럽게 증설될 수 있다.
