# LARS Migration Pipeline

이 워크플로우는 **5개의 전문 에이전트 역할**을 시뮬레이션하여 프로젝트를 체계적으로 완성합니다.

## Phase 0: Product Manager Agent (총괄 기획자)
**목표**: 사용자 요구사항을 정의하고 전체 프로젝트 로드맵을 수립합니다.
1. 사용자의 요청(새로운 기능 등)을 접수합니다.
2. `Requirement_Spec.md` (기획서)를 작성하여 기능의 목적, 주요 흐름(User Flow), 제약 사항을 정의합니다.
3. 작업의 우선순위를 정하고 개발 에이전트들에게 지침을 하달합니다.
> **실행**: "기획해줘" 또는 "PM 모드로 전환해줘"라고 명령하면 이 단계로 진입합니다.

## Phase 1: VBA Analysis Agent (코드 분석가)
**목표**: 기존 VBA 코드를 완벽하게 이해하고 메타데이터를 추출합니다.
1. `VBA/Modules` 폴더의 모든 `.bas`, `.cls` 파일을 스캔합니다.
2. 각 모듈의 **Input/Output** 데이터 구조를 파악합니다.
3. `Analysis_Report.md`를 생성하여 백엔드 에이전트에게 넘길 명세를 작성합니다.
> **실행**: 에이전트에게 `/verify-vba` 명령을 내리면 분석 모드로 진입합니다.

## Phase 2: Backend Architect Agent (백엔드 개발자)
**목표**: 분석된 내용을 바탕으로 C# 코어 로직(Engine)을 구현합니다.
// turbo-all
1. `Features/NewFeature` 폴더를 생성합니다.
2. 분석 리포트를 기반으로 `DTO/Model`을 먼저 정의합니다. (`Models/`)
3. 비즈니스 로직(`Services/`)을 구현하고 단위 테스트(`TestRunner`)를 작성합니다.
4. **검증**: `dotnet run -- test` 명령으로 로직이 정상 동작하는지 확인합니다.

## Phase 3: UI Designer Agent (UI 개발자)
**목표**: 완성된 백엔드 위에 사용자 인터페이스를 씌웁니다.
1. `Backend` 모델의 데이터 타입을 확인합니다.
2. `Forms/` 폴더에 새로운 `UserControl` (예: `NewFeatureControl.cs`)을 생성합니다.
3. 화면에 `DataGridView`와 `Button`을 배치하고, 백엔드 서비스와 연결(Binding)합니다.
4. 디자인(색상, 레이아웃)을 `MainForm` 테마에 맞춥니다.

## Phase 4: Optimization Agent (최적화 전문가)
**목표**: 코드 품질을 높이고, 성능을 최적화하며, 유지보수성을 강화합니다.
1. **코드 리뷰**: 중복된 로직(예: 중복된 파일 읽기 등)을 찾아 유틸리티(`Utils/`)로 통합합니다.
2. **성능 튜닝**: 대용량 파일 처리 시 메모리 사용량을 점검하고 최적화(Stream 사용 등)합니다.
3. **주석 및 문서화**: 누락된 주석을 채우고, 변수명을 직관적으로 재정비합니다.
4. **구조 개선**: `MainForm.cs` 등이 너무 비대해지지 않도록 책임을 분리합니다.

---
**사용 방법**:
이 파일이 있다면, 에이전트에게 **"Phase 1 분석 시작해줘"**, **"최적화해줘"**라고 지시하면 해당 역할에 맞춰 작업을 수행합니다.
